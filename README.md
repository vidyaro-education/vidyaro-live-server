# Vidyaro Live Server

A dedicated **nginx-rtmp** based live-streaming & recording microservice for the Vidyaro edutech platform. It is fully decoupled from the main app and effortlessly deployed using **Coolify**.

## Architecture Overview

```text
Teacher (OBS)
  │  rtmp://<server>:1935/stream
  │  Stream key: <roomId>?key=<streamKey>
  ▼
Nginx (Container via Coolify)
  ├── on_publish  → POST /api/stream-auth (Next.js validates stream logic)
  ├── HLS output  → /tmp/live/<roomId>/index.m3u8 (AES-128 secure)
  ├── FLV record  → /recordings/<roomId>-TIMESTAMP.flv
  │
  └── On Stream End: executes recorder.sh
          │
          ├── ffmpeg converts FLV to web-optimized MP4 
          ├── Uploads to Cloudflare R2
          ├── Determines duration and queries Appwrite for `lectureId`
          └── POST /api/internal/recording-ready alerts Next.js to publish the recording

Students
  └── Watch HLS → https://live.vidyaro.com/live/<roomId>/index.m3u8
                 (AES-128 keys fetched from Next.js, mapped internally)
```

## Coolify Setup Guide

The repository is built natively for Docker Compose deployment on Coolify.

### deployment Steps

1. **Create Resource**: Open Coolify > Projects > Add New Resource > Choose **Docker Compose**.
2. **Repository**: Select this repository (`vidyaro-education/vidyaro-live-server`)
3. **Ports configuration**: 
   - Coolify provides an intuitive UI to route domains. Map your domain `live.vidyaro.com` to port **8080** natively using Coolify's proxy (Traefik/Caddy).
   - In Coolify's Service settings, under "Ports" or under the underlying Server network/firewall, explicitly map and **expose raw TCP port 1935** to the public internet. RTMP needs this port open to accept ingest.
4. **Environment Variables**: Add the variables specified below (found in `.env.example`).
5. **Deploy**: Click the deploy button logic to spin up the single container.

### Environment variables

Copy `.env.example`, change `.env` locally to test or add these straight into Coolify's "Environment Variables" section.

| Variable | Description |
|---|---|
| `STREAM_AUTH_URL` | Next.js API route for stream validation e.g. `https://vidyaro.com/api/stream-auth` |
| `VIDYARO_APP_DOMAIN` | Target domain for CORS and Player requests e.g. `vidyaro.com` |
| `VIDYARO_APP_URL` | Exact URL to app API for recording webhook e.g. `https://vidyaro.com` |
| `INTERNAL_SECRET` | Backend shared secret — restricts unauthorized `/api/hls-key/` retrieval. Generated by `openssl rand -hex 32` |
| `R2_ACCOUNT_ID` | Cloudflare Account Hash |
| `R2_ACCESS_KEY_ID` | Secret Access String Token |
| `R2_SECRET_ACCESS_KEY` | Real token string |
| `R2_BUCKET` | R2 Bucket name e.g. `vidyaro-recordings` |
| `R2_PUBLIC_URL` | Mapped R2 CNAME Domain e.g. `https://recordings.vidyaro.com` |
| `APPWRITE_ENDPOINT` | Appwrite URL instance e.g `https://cloud.appwrite.io/v1` |
| `APPWRITE_PROJECT_ID` | Your specific project ID string |
| `APPWRITE_API_KEY` | Application access token (must enable `databases.read`) |
| `APPWRITE_DATABASE_ID` | Associated DB ID wrapper |
| `APPWRITE_STREAM_KEYS_COLLECTION_ID` | Associated collection wrapper |

## Technical File Tour

Since the repository is refactored, the flow runs smoothly without lingering sidecar components:

- **`docker-compose.yaml`**: The entrypoint configuration read actively by Coolify. Provisions large RAM caches (`tmpfs`) to prevent physical disk grinding against HLS streaming.
- **`Dockerfile`**: Builds Nginx with internal Alpine components including Python, Ffmpeg and AWS-CLI for flawless containerized streaming and processing.
- **`nginx.conf.template`**: Highly secured HTTP Server block configurations. Actively injects the Coolify Environmental variables on boot.
- **`entrypoint.sh`**: Takes variables from container configuration and maps it to Nginx files on launch boot dynamically.
- **`recorder.sh`**: The brain of the operation. Triggered automatically on stream disconnect via `exec_record_done`. Connects FFmpeg stream manipulation natively out to R2 object layers.
