# Vidyaro — Live Server Setup

> **This stack runs OUTSIDE Coolify** using plain `docker compose`.
> Traefik (managed by Coolify) still handles HTTPS — we join the `coolify` Docker network.

---

## File Structure

```
infrastructure/live-server/
  docker-compose.yml      ← the stack definition (rtmp + recorder)
  root-nginx.conf         ← nginx template (contains REPLACE_WITH_INTERNAL_SECRET placeholder)
  nginx.conf              ← generated by deploy.sh — DO NOT edit manually, DO NOT commit
  .env.example            ← copy to .env and fill in your values
  .env                    ← your real credentials — DO NOT commit
  deploy.sh               ← one-shot deploy/update script
  recorder/
    Dockerfile
    watch_and_upload.py
```

## First Deploy (run once)

### 1. Copy files to the server

```bash
# From your local machine
scp -r infrastructure/live-server/ root@76.13.246.64:/opt/vidyaro/live/
```

### 2. Create and fill in .env

```bash
ssh root@76.13.246.64
cd /opt/vidyaro/live

cp .env.example .env
nano .env
# Fill in:
#   INTERNAL_SECRET=ndYVxs30SpvwkLDHVFTb9FffvxDMsilsTiID8knilqh
#   R2_ACCOUNT_ID, R2_ACCESS_KEY_ID, R2_SECRET_ACCESS_KEY
#   R2_BUCKET_NAME=vidyaro-lectures
#   R2_PUBLIC_URL=https://lectures-cdn.vidyaro.com
#   APPWRITE_ENDPOINT=https://api.vidyaro.com/v1
#   APPWRITE_PROJECT_ID, APPWRITE_API_KEY
#   APPWRITE_DATABASE_ID=vidyaro_db
#   APPWRITE_LECTURES_COL_ID=lectures
#   VIDYARO_APP_URL=https://vidyaro.com
```

### 3. Remove the Coolify-managed rtmp container

```bash
# Stop and remove the old Coolify-managed container
docker rm -f rtmp-fwgkkw8s80gg88c0wowwgoog 2>/dev/null || true
docker rm -f vidyaro-recorder 2>/dev/null || true

# Also disable it in Coolify UI so it doesn't get restarted:
# Coolify → your live server service → click "Stop" or delete the service
```

### 4. Deploy

```bash
cd /opt/vidyaro/live
chmod +x deploy.sh
./deploy.sh
```

That's it. The script:
- Reads `INTERNAL_SECRET` from `.env`
- Generates `nginx.conf` with the secret injected
- Builds the recorder Docker image
- Starts both containers

## Updating

### Update nginx.conf only (no recorder downtime)

```bash
# From local machine
scp infrastructure/live-server/root-nginx.conf root@76.13.246.64:/opt/vidyaro/live/

# On the server
cd /opt/vidyaro/live
./deploy.sh nginx
```

### Update the recorder script

```bash
# From local machine
scp infrastructure/live-server/recorder/watch_and_upload.py root@76.13.246.64:/opt/vidyaro/live/recorder/

# On the server
cd /opt/vidyaro/live
./deploy.sh update
```

### Full redeploy (both services)

```bash
cd /opt/vidyaro/live
./deploy.sh update
```

## Make it survive reboots (optional but recommended)

```bash
# Create a systemd service so docker compose starts on server reboot
cat > /etc/systemd/system/vidyaro-live.service << 'EOF'
[Unit]
Description=Vidyaro Live Server Stack
Requires=docker.service
After=docker.service network-online.target

[Service]
Type=oneshot
RemainAfterExit=yes
WorkingDirectory=/opt/vidyaro/live
ExecStart=/usr/bin/docker compose up -d
ExecStop=/usr/bin/docker compose down
TimeoutStartSec=120

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable vidyaro-live
systemctl start vidyaro-live
```

## How the full system works

```
OBS → rtmp://live.vidyaro.com/stream/{lectureId}?pwd=SECRET
         ↓
    [nginx-rtmp :1935]
         ├── HLS segments → /tmp/live/{lectureId}/ → Traefik → students
         ├── AES-128 keys → /recordings/keys/       → Next.js /api/hls-key/
         └── FLV recording → /recordings/
                                  ↓
                     [vidyaro-recorder] (polls every 5s)
                                  ↓
                     ffmpeg → MP4 (fast remux)
                                  ↓
                     aws s3 cp → Cloudflare R2
                                  ↓
                     Appwrite PATCH lecture.videoUrl
```

### Anti-piracy layers

| Layer | Mechanism |
|---|---|
| Rate limit | 30 req/s per IP in nginx — blocks yt-dlp/ffmpeg scrapers |
| Referer lock | `valid_referers vidyaro.com *.vidyaro.com` — blocks embedding elsewhere |
| AES-128 encryption | `.ts` segments are unplayable without the key |
| Key gating | Keys served by Next.js only after JWT auth + 2-device check |
| 2-device limit | Redis sorted-set concurrent session limiter |

## Troubleshooting

**nginx won't start:**
```bash
docker logs vidyaro-live
# Common issues:
#  - nginx.conf not generated (run ./deploy.sh again)
#  - Port 1935 already in use (docker rm -f old-rtmp-container)
```

**Recorder not processing .done files:**
```bash
docker logs vidyaro-recorder
ls -la /opt/vidyaro/recordings/  # check for .done and .flv files
```

**HLS key 403 from nginx:**
```bash
grep "ndYVxs30" /opt/vidyaro/live/nginx.conf  # must show your actual secret
# If still REPLACE_WITH_INTERNAL_SECRET, run ./deploy.sh again
```
